# logi-canvus 画面設計書

## 画面一覧

| # | 画面名 | URL | 認証 |
|---|--------|-----|------|
| 1 | ログイン | `/login` | 不要 |
| 2 | アカウント作成 | `/signup` | 不要 |
| 3 | 招待受け入れ | `/invite/[token]` | 不要 |
| 4 | ダッシュボード | `/[org-slug]` | 必要 |
| 5 | ページエディタ | `/[org-slug]/[page-slug]` | 必要 |
| 6 | 組織設定 | `/[org-slug]/settings` | 必要 |

---

## App Router ルート構成

```
app/
  (auth)/                          ← 未認証レイアウト（サイドバーなし）
    login/
      page.tsx
    signup/
      page.tsx
    invite/
      [token]/
        page.tsx
  (app)/                           ← 認証済みレイアウト（サイドバーあり）
    [org-slug]/
      page.tsx                     ← ダッシュボード
      settings/
        page.tsx                   ← 組織設定（静的セグメントが [page-slug] に優先）
      [page-slug]/
        page.tsx                   ← ページエディタ（シェルはSSR）
        loading.tsx
        error.tsx
```

> `settings` は静的セグメントのため `[page-slug]` より優先される。Route Groups で認証要否をLayout単位で分離する。

## 画面遷移図

```
[未認証]
  ├── /login ──────────────────────────────→ /[org-slug]（ダッシュボード）
  ├── /signup
  │     ├── Step1: ユーザー情報入力
  │     └── Step2: 組織作成 or 招待コード入力 → /[org-slug]
  └── /invite/[token]
        ├── 未登録 → /signup?redirect=/invite/[token]
        └── 登録済み → /login?redirect=/invite/[token]
                       → ログイン後トークンURLに戻り → 組織参加 → /[org-slug]

[認証済み]
  /[org-slug]（ダッシュボード）
    ├── ページ選択          → /[org-slug]/[page-slug]
    ├── 新規ページ作成      → /[org-slug]/[page-slug]（APIレスポンスのslugで遷移）
    └── 組織設定            → /[org-slug]/settings

  /[org-slug]/[page-slug]（ページエディタ）
    ├── サイドバーでページ切替
    ├── アノテーションパネル開閉（オーバーレイ）
    └── 競合発生時 → 競合解決ダイアログ（モーダル）
```

## 各画面詳細

---

### 1. ログイン画面 `/login`

**目的：** 既存ユーザーの認証

**レイアウト：**
```
┌─────────────────────────────┐
│         logi-canvus         │
│                             │
│   [Email input]             │
│   [Password input]          │
│                             │
│   [ログインボタン]           │
│                             │
│   アカウントをお持ちでない方  │
│   → アカウント作成           │
└─────────────────────────────┘
```

**UI要素：**
- Emailフィールド
- Passwordフィールド
- ログインボタン
- アカウント作成へのリンク

**遷移：**
- 成功 → `?redirect` パラメータがあればそのURL、なければ `/[org-slug]`
- アカウント作成リンク → `/signup`

---

### 2. アカウント作成画面 `/signup`

**目的：** 新規ユーザー登録 + 組織の作成または参加

**2ステップ構成：**

#### Step 1：ユーザー情報入力
```
┌─────────────────────────────┐
│    アカウント作成 (1/2)      │
│                             │
│   [名前 input]              │
│   [Email input]             │
│   [Password input]          │
│   [Password確認 input]      │
│                             │
│   [次へ]                    │
└─────────────────────────────┘
```

#### Step 2：組織の選択
```
┌─────────────────────────────┐
│    アカウント作成 (2/2)      │
│                             │
│  ○ 新しい組織を作成する      │
│    [組織名 input]            │
│                             │
│  ○ 既存の組織に参加する      │
│    [招待コード input]         │
│                             │
│   [登録する]                 │
└─────────────────────────────┘
```

> `/invite/[token]` 経由の場合は招待コードが自動入力され、Step2をスキップして登録完了。

**遷移：**
- 成功 → `/[org-slug]`
- ログインへのリンク → `/login`

---

### 3. 招待受け入れ画面 `/invite/[token]`

**目的：** 招待リンク経由での組織参加

**レイアウト：**
```
┌─────────────────────────────┐
│                             │
│  「○○組織」に招待されています  │
│                             │
│  [アカウント作成して参加]    │  ← 未登録の場合
│  [ログインして参加]          │  ← 登録済みの場合
│                             │
└─────────────────────────────┘
```

**遷移：**
- 未登録 → `/signup?redirect=/invite/[token]`（登録完了後にトークンURLへ戻り組織参加）
- 登録済み → `/login?redirect=/invite/[token]`（ログイン後にトークンURLへ戻り組織参加）
- トークン無効 → エラーメッセージ表示

> トークンはURLに保持し続けることで、認証フロー中に消失しない。

### 4. ダッシュボード画面 `/[org-slug]`

**目的：** フォルダ・ページの一覧表示と操作

**レイアウト：**
```
┌──────────┬──────────────────────────────────────┐
│サイドバー │  メインエリア                         │
│          │                                      │
│ 組織名   │  最近編集したページ                    │
│          │  ┌──────────┐  ┌──────────┐          │
│ + 新規   │  │ページ名   │  │ページ名   │          │
│          │  │更新日時   │  │更新日時   │          │
│▼ フォルダA│  └──────────┘  └──────────┘          │
│  └ ページ│                                      │
│▼ フォルダB│  フォルダ                             │
│  └ ページ│  📁 フォルダA                         │
│  └ ページ│  📁 フォルダB                         │
│          │                                      │
│──────────│                                      │
│🔔 通知   │                                      │
│⚙ 設定   │                                      │
└──────────┴──────────────────────────────────────┘
```

**Empty State（ページが0件の場合）：**
```
┌──────────┬──────────────────────────────────────┐
│サイドバー │                                      │
│          │   まだページがありません               │
│          │                                      │
│          │   [+ 最初のページを作成する]           │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

**UI要素：**
- サイドバー：フォルダ・ページのツリー表示（折りたたみ可能）
- 新規ページ作成ボタン（`+`）
- 新規フォルダ作成ボタン
- 通知アイコン（未読バッジ付き）
- 組織設定リンク
- メインエリア：最近のページカード + フォルダ一覧
- Empty State：CTA付き

**遷移：**
- ページカード / サイドバーのページ → `/[org-slug]/[page-slug]`
- 新規ページ作成 → APIが生成した `page-slug` を受け取り `/[org-slug]/[page-slug]` へ遷移
- 組織設定 → `/[org-slug]/settings`
- 通知アイコン → 通知ドロップダウン

### 5. ページエディタ画面 `/[org-slug]/[page-slug]`

**目的：** 仕様ドキュメントの作成・編集

**レイアウト：**
```
┌──────────┬────────────────────────────┬───────────────┐
│サイドバー │  エディタエリア              │アノテーション  │
│（4と共通）│                            │パネル         │
│          │  # ページタイトル           │（オーバーレイ）│
│          │                            │               │
│          │  [見出しブロック]           │ 📌 ピン①      │
│          │  [テキストブロック]         │  └ コメント   │
│          │  [テキストブロック]         │  └ 返信...    │
│          │                            │  [解決済みにする]│
│          │  ┌──────────────────────┐  │               │
│          │  │  図ブロック（固定高さ）│  │ 📌 ピン②      │
│          │  │  [Screen]→[Branch]   │  │  └ コメント   │
│          │  │       ↓        ↓     │  │               │
│          │  │  [Action]  [Screen]  │  │               │
│          │  └──────────────────────┘  │               │
│          │                            │               │
│          │  [テキストブロック]         │               │
│          │                            │               │
│──────────│────────────────────────────│───────────────│
│🔔 通知   │  最終保存: 2分前  [保存中]  │               │
└──────────┴────────────────────────────┴───────────────┘
```

**UI要素：**

*ヘッダー / ステータスバー*
- ページタイトル（インライン編集）
  - Enterキーでエディタ先頭ブロックにフォーカス移動
- 保存状態インジケーター（保存中 / 保存済み / 競合）

*エディタエリア（クライアントコンポーネント）*
- ブロック追加メニュー（`/` コマンド or `+` ボタン）
  - テキストブロック
  - 見出しブロック（H1 / H2 / H3）
  - 図ブロック
- ブロックのドラッグ&ドロップで並び替え
- テキストブロック：選択時にツールバー表示（太字・箇条書き等）
- アノテーションピン：テキスト選択 → ピン追加ボタン → 右パネルを自動展開し該当スレッドにスクロール・フォーカス

*図ブロック内*
- キャンバス高さは固定（ノード数に関わらず一定）、ノードが多い場合はキャンバス内でパン操作
- ノードパレット（キャンバス上部ツールバー）：Screen / Action / Branch / Start / End / External
- キャンバス：ノードをドラッグして配置、エッジをドラッグして接続
- ノード上にピン追加ボタン → クリックで右パネルを自動展開し該当スレッドにフォーカス

*アノテーションパネル（右・オーバーレイ）*
- エディタエリアに重なる形で絶対配置（エディタ幅のリフローなし）
- ピン一覧（未解決 / 解決済み切り替えトグル）
  - スコープ：ページ全体（図ブロック・テキストブロック問わず一覧）
- 各ピン：コメントスレッド + 返信フォーム + Resolvedボタン
- Resolvedになったピンはデフォルト非表示（右パネル上部のトグルで表示切り替え）

**page-slug 生成ルール**
- 新規ページ作成時にAPI側で自動生成（例：`login-spec-a1b2c3`）
- タイトル（URLセーフ変換）＋ランダムサフィックスで一意性を保証
- フロントエンドはAPIレスポンスの `url_slug` を受け取ってから `router.push` する

**SSR / CSR 境界**
- `[page-slug]/page.tsx`：Server Component（ページメタデータ取得・OGタグ生成・`<Suspense>` ラッパー）
- `<PageEditor>` コンポーネント：`"use client"` でエクスポート（TipTap / React Flow はCSRのみ）

**TipTap × React Flow データ責務**
- TipTapのドキュメントには `diagramBlockId` のみ保持し、ノード・エッジデータはTipTapのdocに持たせない
- 図データ（DiagramNode / DiagramEdge）はDBを正とし、React Flow の変更はZustandの図専用storeに蓄積してデバウンスでAPIに反映
- これによりTipTapのブロックドラッグとReact Flowのノードドラッグのイベント競合を分離する

**遷移：**
- 競合発生 → 競合解決ダイアログ（モーダル）
- 通知アイコン → 通知ドロップダウン

---

#### 5-a. 競合解決ダイアログ（モーダル）

**目的：** 楽観的ロックによる競合を解決する

```
┌─────────────────────────────────────────┐
│  ⚠️ 編集が競合しました                   │
│                                         │
│  別のメンバーが先に保存しました。          │
│  どちらの変更を採用しますか？             │
│                                         │
│  ┌──────────────┐  ┌──────────────┐    │
│  │ あなたの変更  │  │ 最新の内容    │    │
│  │              │  │              │    │
│  │ （diff表示）  │  │ （diff表示）  │    │
│  │              │  │              │    │
│  └──────────────┘  └──────────────┘    │
│                                         │
│  [あなたの変更を採用]  [最新の内容を採用] │
└─────────────────────────────────────────┘
```

**diff表示の仕様：**
- 競合検出の単位：**ブロック単位**（`Block.version` で検出）
- 変更のあったブロックのみ左右に並べて表示
- テキストブロック：文字レベルのdiff（追加を緑・削除を赤でハイライト）
- 図ブロック：「ノードが追加された」「エッジが削除された」等の人間可読な差分テキストで表示
- 「あなたの変更を採用」時：フロントエンドは最新の `Block.version` を取得してから上書きPATCHを送信

---

#### 5-b. 通知ドロップダウン

**目的：** ツール内通知の確認

```
┌──────────────────────────────┐
│ 通知                    全て既読│
│                              │
│ 🔵 田中さんがピンに返信しました  │
│    ログイン機能 仕様 · 2分前    │
│                              │
│ 🔵 山田さんがメンションしました  │
│    決済フロー仕様 · 1時間前     │
│                              │
│ ⚪ 編集が競合しました           │
│    ユーザー登録フロー · 昨日    │
│                              │
└──────────────────────────────┘
```

**通知種別：**
| 種別 | 内容 |
|------|------|
| `annotation_reply` | 自分のピンにコメントが返信された |
| `mention` | コメント内で自分がメンションされた |
| `conflict` | 楽観的ロックの競合が発生した |

### 6. 組織設定画面 `/[org-slug]/settings`

**目的：** 組織情報の管理・メンバーの招待・組織からの脱退

**レイアウト：**
```
┌──────────┬──────────────────────────────────────┐
│サイドバー │  組織設定                             │
│（共通）  │                                      │
│          │  ## 組織情報                          │
│          │  組織名: [input]          [保存]      │
│          │                                      │
│          │  ## メンバー                          │
│          │  ┌──────────────────────────────┐    │
│          │  │ 名前      メール             │    │
│          │  │ 田中太郎  tanaka@...         │    │
│          │  │ 山田花子  yamada@... （自分） │    │
│          │  │                  [組織を脱退] │    │
│          │  └──────────────────────────────┘    │
│          │                                      │
│          │  ## メンバー招待                      │
│          │  [メールアドレス input]  [招待を送る]  │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

**権限設計：**
- メンバーの削除操作は不可（フラット権限のため他ユーザーを除名する権限は誰も持たない）
- 自分自身のみ「組織を脱退」ボタンで脱退可能
- 脱退後、自分が `created_by` のページ・アノテーションは残存する（削除されない）
- 脱退確認モーダルを表示してから実行

**その他のUI要素：**
- 組織名編集フォーム
- メンバー一覧（名前・メール・自分のみ脱退ボタン）
- 招待フォーム（メールアドレス入力 → 招待メール送信）

## 共通コンポーネント

### サイドバー
- 全認証画面で共通表示
- フォルダ：折りたたみ可能ツリー
- ページ：クリックで遷移
- 右クリックメニュー：名前変更 / 削除（確認モーダルあり） / フォルダへ移動
- 通知アイコン（未読数バッジ）
- 組織設定リンク

### 削除確認モーダル
ページ・フォルダ削除時に表示。削除は不可逆であることを明示する。
```
┌─────────────────────────────┐
│  「○○」を削除しますか？      │
│                             │
│  この操作は元に戻せません。   │
│                             │
│  [キャンセル]  [削除する]    │
└─────────────────────────────┘
```

### ブロック追加メニュー
- `/` キーまたはブロック左の `+` ボタンで表示
- 選択肢：テキスト / 見出しH1・H2・H3 / 図ブロック
